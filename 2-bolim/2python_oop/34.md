## 1. Kirish va Inkapsulatsiya tushunchasi (20 daqiqa)

Assalomu alaykum, yosh dasturchilar! üòä Bugun biz Python‚Äôda OOP (Obyektga yo‚Äònaltirilgan dasturlash) doirasida **Inkapsulatsiya (Encapsulation)** tushunchasini o‚Äòrganamiz. Inkapsulatsiya ‚Äî bu sinf ichidagi ma‚Äôlumotlarni himoya qilish va tashqi foydalanuvchilardan yashirish jarayoni. Bugun public, protected va private atributlar bilan ishlashni ko‚Äòrib chiqamiz.

**Inkapsulatsiya nima?**  
Inkapsulatsiya ‚Äî bu sinf ichidagi atributlar va metodlarni bir butun sifatida birlashtirish va ularga kirishni cheklash orqali ma‚Äôlumotlarni himoya qilish. Bu OOPning asosiy tamoyillaridan biri bo‚Äòlib, quyidagi afzalliklarga ega:  
- Ma‚Äôlumotlarni tasodifiy o‚Äòzgartirishdan himoya qilish.  
- Sinf ichki tuzilmasini tashqi foydalanuvchilardan yashirish.  
- Kodni yanada xavfsiz va tartibli qilish.  

Python‚Äôda atributlarning kirish darajalari:  
- **Public**: Hamma joydan foydalanish mumkin (masalan, `self.ism`).  
- **Protected**: Faqat sinf va uning voris sinflari ichida ishlatiladi (masalan, `self._ism`).  
- **Private**: Faqat sinf ichida ishlatiladi (masalan, `self.__ism`).  

Oddiy misol bilan boshlaymiz:

```python
class Shaxs:
    def __init__(self, ism):
        self.ism = ism  # Public atribut

shaxs = Shaxs("Ali")
print(shaxs.ism)  # Ali
shaxs.ism = "Vali"  # Tashqaridan o‚Äòzgartirish mumkin
print(shaxs.ism)  # Vali
```

Bu yerda `ism` ‚Äî public atribut, uni tashqaridan o‚Äòzgartirish mumkin. Ammo bu xavfsiz emas, shuning uchun inkapsulatsiyadan foydalanamiz.

Keling, bir savol: Inkapsulatsiya nima uchun kerak deb o‚Äòylaysiz? (O‚Äòquvchilar javobini kutaman, to‚Äòg‚Äòri javob bo‚Äòlsa: ‚ÄúAjoyib, to‚Äòg‚Äòri!‚Äù deyman.)

Mashq: Quyidagi vazifani bajaring:  
- `Talaba` nomli sinf yarating, u `ism` nomli public atributga ega bo‚Äòlsin.  
- Bitta talaba obyekti yarating, uning `ism` atributini o‚Äòzgartirib, natijani ekranga chiqaring.

## 2. Public va Protected atributlar (25 daqiqa)

Endi public va protected atributlarni chuqurroq o‚Äòrganamiz! Bu atributlar sinf ichidagi ma‚Äôlumotlarga kirishni boshqarishda ishlatiladi.

### Public atributlar
Public atributlar sinf ichida va tashqarisida erkin foydalanish uchun ochiq bo‚Äòladi:

```python
class Kitob:
    def __init__(self, nomi, muallif):
        self.nomi = nomi  # Public atribut
        self.muallif = muallif  # Public atribut

    def malumot(self):
        return f"Kitob: {self.nomi}, Muallif: {self.muallif}"

kitob = Kitob("Alpomish", "Xalq ertagi")
print(kitob.nomi)  # Alpomish
kitob.nomi = "O‚Äòtgan kunlar"  # Tashqaridan o‚Äòzgartirish
print(kitob.malumot())  # Kitob: O‚Äòtgan kunlar, Muallif: Xalq ertagi
```

Public atributlar hamma uchun ochiq bo‚Äòlgani uchun ularga ehtiyotkorlik bilan ishlash kerak.

### Protected atributlar
Protected atributlar `_` (bitta pastki chiziq) bilan boshlanadi va faqat sinf ichida yoki voris sinflarda ishlatilishi tavsiya etiladi. Ammo Python‚Äôda bu qoida majburiy emas, faqat konvensiya sifatida qabul qilinadi:

```python
class Shaxs:
    def __init__(self, ism, yosh):
        self._ism = ism  # Protected atribut
        self._yosh = yosh  # Protected atribut

    def malumot(self):
        return f"Ism: {self._ism}, Yosh: {self._yosh}"

class Talaba(Shaxs):
    def toliq_malumot(self):
        return f"Talaba: {self._ism}, {self._yosh} yoshda."

shaxs = Shaxs("Ali", 20)
print(shaxs.malumot())  # Ism: Ali, Yosh: 20
talaba = Talaba("Zilola", 22)
print(talaba.toliq_malumot())  # Talaba: Zilola, 22 yoshda.

# Tashqaridan kirish (tavsiya etilmaydi, lekin mumkin)
print(shaxs._ism)  # Ali
```

Protected atributlar voris sinflar uchun ochiq bo‚Äòladi, lekin tashqi foydalanuvchilar ularga to‚Äòg‚Äòridan-to‚Äòg‚Äòri kirmasligi kerak.

### Public va Protected farqi
Public atributlar hamma uchun ochiq bo‚Äòlsa, protected atributlar sinf va uning vorislariga mo‚Äòljallangan:

```python
class Mashina:
    def __init__(self, model, tezlik):
        self.model = model  # Public atribut
        self._tezlik = tezlik  # Protected atribut

    def malumot(self):
        return f"Model: {self.model}, Tezlik: {self._tezlik} km/soat"

mashina = Mashina("Chevrolet", 120)
print(mashina.model)  # Chevrolet
print(mashina._tezlik)  # 120 (tavsiya etilmaydi)
print(mashina.malumot())  # Model: Chevrolet, Tezlik: 120 km/soat
```

Protected atributlar ma‚Äôlumotlarni qisman himoya qiladi.

Mashq: Quyidagi vazifani bajaring:  
- `Hayvon` nomli sinf yarating, u `turi` (public) va `_yosh` (protected) atributlariga ega bo‚Äòlsin.  
- `malumot` nomli metod yozing, u hayvon haqida ma‚Äôlumot qaytarsin.  
- Bitta hayvon obyekti yarating va uning atributlarini ekranga chiqaring.  

5 daqiqa vaqt beraman!

## 3. Private atributlar bilan ishlash (20 daqiqa)

Ajoyib, endi private atributlarni o‚Äòrganamiz! Private atributlar sinf ichida to‚Äòliq himoyalangan bo‚Äòladi.

### Private atributlar
Private atributlar `__` (ikki pastki chiziq) bilan boshlanadi va faqat sinf ichida ishlatiladi. Tashqaridan ularga to‚Äòg‚Äòridan-to‚Äòg‚Äòri kirish mumkin emas:

```python
class Shaxs:
    def __init__(self, ism, yosh):
        self.__ism = ism  # Private atribut
        self.__yosh = yosh  # Private atribut

    def malumot(self):
        return f"Ism: {self.__ism}, Yosh: {self.__yosh}"

shaxs = Shaxs("Ali", 20)
print(shaxs.malumot())  # Ism: Ali, Yosh: 20
# print(shaxs.__ism)  # Xato: AttributeError
```

Private atributlar tashqaridan ko‚Äòrinmaydi, lekin Python‚Äôda ularni ‚Äúname mangling‚Äù orqali kirish mumkin (lekin bu tavsiya etilmaydi):

```python
print(shaxs._Shaxs__ism)  # Ali (tavsiya etilmaydi)
```

### Getter va Setter metodlari
Private atributlar bilan ishlash uchun getter va setter metodlaridan foydalanamiz:

```python
class Talaba:
    def __init__(self, ism, baho):
        self.__ism = ism
        self.__baho = baho

    # Getter metod
    def get_baho(self):
        return self.__baho

    # Setter metod
    def set_baho(self, yangi_baho):
        if 0 <= yangi_baho <= 100:
            self.__baho = yangi_baho
            return "Baho o‚Äòzgartirildi."
        return "Baho 0-100 oralig‚Äòida bo‚Äòlishi kerak."

    def malumot(self):
        return f"Talaba: {self.__ism}, Baho: {self.__baho}"

talaba = Talaba("Zilola", 85)
print(talaba.malumot())  # Talaba: Zilola, Baho: 85
print(talaba.get_baho())  # 85
print(talaba.set_baho(90))  # Baho o‚Äòzgartirildi.
print(talaba.malumot())  # Talaba: Zilola, Baho: 90
print(talaba.set_baho(150))  # Baho 0-100 oralig‚Äòida bo‚Äòlishi kerak.
```

### Private atributlarning afzalligi
Private atributlar ma‚Äôlumotlarni to‚Äòliq himoya qiladi va ularga faqat sinf ichidagi metodlar orqali kirish mumkin. Bu xavfsizlikni oshiradi.

Mashq: Quyidagi vazifani bajaring:  
- `Mashina` nomli sinf yarating, u `__model` va `__tezlik` nomli private atributlarga ega bo‚Äòlsin.  
- `get_tezlik` va `set_tezlik` metodlarini yozing (tezlik 0-300 oralig‚Äòida bo‚Äòlsin).  
- Bitta mashina obyekti yarating, tezlikni o‚Äòzgartirib, natijani ekranga chiqaring.
