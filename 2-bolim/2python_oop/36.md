## 2. property() dekoratori bilan ishlash (25 daqiqa)

Endi `@property` dekoratori bilan tanishamiz! Bu dekorator getter va setter metodlarini atribut sifatida ishlatish imkonini beradi.

### @property dekoratori nima?
`@property` dekoratori private atributlarga toâ€˜gâ€˜ridan-toâ€˜gâ€˜ri kirishni taqlid qiladi, lekin ichki logikani saqlab qoladi. Bu getter va setter metodlarini yanada qulay qiladi:

```python

        self.__ism = ism
        self.__baho = baho

    @property
    def baho(self):  # Getter
        return self.__baho

    @baho.setter
    def baho(self, yangi_baho):  # Setter
        if 0 <= yangi_baho <= 100:
            self.__baho = yangi_baho
        else:
            print("Baho 0-100 oraligâ€˜ida boâ€˜lishi kerak!")

talaba = Talaba("Zilola", 85)
print(talaba.baho)  # 85 (get_ism() oâ€˜rniga atribut sifatida ishlatildi)
talaba.baho = 90  # set_ism() oâ€˜rniga atribut sifatida ishlatildi
print(talaba.baho)  # 90
talaba.baho = 150  # Baho 0-100 oraligâ€˜ida boâ€˜lishi kerak!
```

### @property dekoratorining afzalliklari
- Kodni soddalashtiradi: `talaba.get_baho()` oâ€˜rniga `talaba.baho` yozish mumkin.  
- Maâ€™lumotlarni himoya qilishni davom ettiradi.  
- Tashqi foydalanuvchilar uchun atributga oâ€˜xshab koâ€˜rinadi, lekin ichki logika saqlanadi.
